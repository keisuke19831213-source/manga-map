# MANGA MAP — マンガの歴史とジャンルの進化マップ

[musicmap.info](https://musicmap.info) のマンガ版。マンガのジャンルがどのように生まれ、
影響し合い、進化してきたかを一枚のインタラクティブマップで可視化するプロジェクトです。

## 機能

- **ジャンルマップ** (`/`) — 1900年から現在までの45ジャンルと影響関係を系統樹として表示。
  ドラッグで移動、ホイールで拡大縮小。ノードをクリックすると解説・代表作・関連ジャンルを表示。
  カテゴリ(少年/少女/青年/ギャグ/オルタナ/デジタル)でのフィルタも可能。
- **作品図鑑** (`/works`) — マンガ史の節目となった38作品のカタログ。カテゴリで絞り込み。
- **作品ページ** (`/works/[id]`) — 作品解説に加えて、ユーザーが投稿できる:
  - **おすすめコメント** — この作品を推す声
  - **コマ語り** — 「◯巻◯ページの◯コマ目」をピンポイントで指定して語るコメント
- **マンガ史年表** (`/history`) — 約120年のマンガ史を9つの時代で解説。
- **みんなの投稿** (`/community`) — 全投稿のタイムライン。図鑑にない作品のおすすめも自由入力で投稿可能。

## 起動方法

```bash
npm install
npm run dev   # http://localhost:3100
```

## 仕組み

- Next.js (App Router) + TypeScript。外部依存は最小限。
- ジャンル・影響関係・作品のマスターデータは `lib/data.ts` に集約。
  ノードやエッジを追加したいときはこのファイルを編集するだけでマップに反映されます。
- ユーザー投稿は `data/posts.json` にファイル保存(API: `app/api/posts/route.ts`)。
  本格運用時はここをDB(SQLite/Postgres等)に差し替えるのが次のステップです。

## データの追加方法

- **ジャンルを増やす**: `lib/data.ts` の `GENRES` にノードを追加し、`EDGES` に影響関係を追加。
  `year` で縦位置、`cat` で横位置(列)が自動で決まります。重なる場合は `ly`(レイアウト用年)で微調整。
- **作品を増やす**: `WORKS` に追加。`genres` に該当ジャンルidを入れるとマップの詳細パネルと図鑑に自動で出ます。
